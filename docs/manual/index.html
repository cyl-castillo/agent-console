User Manual

Everything you need to install Agent Console and drive a coding agent through real work — without losing control of your repository.

What is Agent Console

Agent Console is a minimalist, terminal-first desktop app for directing AI agents inside a repository. It pairs the Claude Code CLI with an integrated shell, a git diff viewer, and per-turn snapshots, so you can hand real work to an agent and verify every step.

It is deliberately not an IDE and not a chat client. The center of the app is the chat, the terminal, and the git diff — not a code editor. You don't read code here; you direct the agent and check its work.

┌──────────────────────────────────────────────────────────────────────┐ │ my-repo · javascript · vite · /home/me/code/my-repo │ ├──────────┬─────────────────────────────────────┬─────────────────────┤ │ Files │ [Terminal] [Changes (3)] │ Agent │ │ │ ─────────────────────────────────── │ ─────────────────── │ │ src/ │ $ npm test │ you ↶restore │ │ api.ts │ ✓ 14 passed │ "fix the failing │ │ app.ts │ │ user-auth test" │ │ tests/ │ │ ▸ Read tests/auth │ │ README │ │ ✓ Edit src/api.ts │ │ │ │ ✓ Bash: npm test │ │ │ │ claude: fixed — │ │ │ │ token was expired │ │ │ │ [type a task…] │ └──────────┴─────────────────────────────────────┴─────────────────────┘

Principles

  • Terminal-first, agent-first, diff-first.
  • Lightweight — built on Tauri + Rust, not Electron.
  • Cross-platform by design (Linux / macOS / Windows), Linux-first in practice.
  • Human control mandatory before risky actions.
  • Local-first — no accounts, no cloud sync, no telemetry, no marketplace.
Status · early preview Single user, local only, no auto-update. Suitable for daily personal use and demos. APIs and event names may still change between minor releases.

Install & first run

  1. Authenticate the CLI. Run claude in any terminal once and complete login. Agent Console reuses that session — it never asks for your API key.
  2. Launch Agent Console. Install a release from the downloads on the home page (.deb, .AppImage, or .rpm on Linux; .dmg / .msi on macOS / Windows), or build from source (see below).
  3. Open a repository. Use the folder picker or pick from the recents list. Any local folder works; a git repo unlocks the diff viewer and snapshots.
  4. Give it a task. Type a plain-language instruction into the chat — e.g. "fix the failing user-auth test" — and press Enter.
  5. Approve as it goes. Each time the agent wants to edit a file or run a command, a modal asks you to approve or deny.
No accounts There is nothing to sign up for inside Agent Console. It runs entirely on your machine and uses your existing Claude Code login.

The interface

The window is three columns. From left to right:

◧ Files

The file tree of the open repository. Browse structure at a glance — this is for orientation, not editing.

▤ Terminal / Changes / Preview

The work area, tabbed. A real PTY terminal, the git diff viewer, and a preview pane. Switch with Ctrl+1/2/3.

◨ Agent

The chat. You type tasks here and watch the agent's tool calls and replies stream in, each with a restore handle.

Two sidebars can be toggled to give the work area more room: Ctrl+B hides the workspace sidebar, Ctrl+J toggles the side panel.

When the agent first mutates files in a turn, the work area auto-switches to the Changes tab so you see the diff without hunting for it.

Directing the agent

The chat panel runs claude -p inside your repository with full tool access. You don't write code or call tools yourself — you describe the outcome and supervise.

A good loop

  1. State the task in plain language. Be specific about the goal and how to verify it ("…and make npm test pass").
  2. Watch the tool calls stream: Read, Edit, Bash, etc. Safe reads pass through; mutations pause for your approval.
  3. Read the agent's summary at the end of the turn. Check the Changes tab and run the tests yourself in the terminal.
  4. Follow up in the same chat to refine, or restore if the turn went sideways.

Assistant text is rendered as Markdown — headings, lists, inline code, and code blocks all display properly. Press Ctrl+K any time to jump focus to the chat input; Shift+Enter adds a newline without sending.

Terminal & sessions

The Terminal tab is a full PTY — it runs npm test, vim, git status, anything you'd run in a normal shell. It's how you verify the agent's work with your own hands.

Sessions

Each session is an independent live context. Create a new one with Ctrl+T and cycle between live sessions with Ctrl+Tab / Ctrl+Shift+Tab (or Ctrl+] / Ctrl+[). Close the active one with Ctrl+W. Your recent projects persist across launches.

Focus-aware shortcuts Some shortcuts pass through to the shell when your cursor is inside the terminal — so Ctrl+T, Ctrl+W, Ctrl+L, and Ctrl+[ reach the program you're running (e.g. vim) instead of the app. Click outside the terminal first if you want the app action.

Reviewing changes

The Changes tab is a git diff viewer. Every file the agent touches shows up with a status badge:

BadgeMeaning
MModified — existing tracked file changed.
AAdded — new file the agent created.
DDeleted — file removed.
UUntracked — present on disk, not yet in git.

You can revert a single file or revert everything with one click. Refresh the view manually with Ctrl+R if you've made changes outside the app.

The safety model

Agent Console assumes the agent will sometimes be wrong. Two independent layers keep you in control, and both are on by default:

1 · Tool approval — the front line

You approve every mutation before it happens.

2 · Per-turn snapshot — the safety net

One click rewinds the whole working tree if a turn goes wrong.

Tool approvals

Before the agent runs a state-changing tool, a modal shows you the tool name and its exact input, and you approve or deny.

Approval is required for: Bash, Edit, Write, MultiEdit, and NotebookEdit. Read-only tools — Read, Grep, Glob, LS, WebFetch, WebSearch — pass through without interrupting you.

  • Ctrl+Enter approves the pending action.
  • Esc denies it (and dismisses any modal).

If you trust the current task, a session-wide "approve all" toggle lets the agent run uninterrupted for that session.

How it works Approvals are enforced by a bundled PreToolUse hook (a small Node script) that Claude Code runs before each tool. This is why Node.js is a runtime requirement.

Per-turn snapshots

Right before each message you send is dispatched, the app builds a git commit capturing the full working tree — tracked and untracked files — under a hidden ref (refs/agent-console/<turn-id>).

If a turn goes sideways, click ↶ restore next to your message bubble. The app resets the working tree back to that exact state.

Your branches are safe Restore uses read-tree --reset -u — it rewrites the working tree only. HEAD is never moved and your branches are never touched. The snapshot refs live in their own namespace, out of the way of your normal git history.
Non-git folders For a folder that isn't a git repository there is no snapshot to take, so only the approval layer applies. Open a git repo to get the full safety net.

Build from source

If you'd rather build it yourself, you'll need the Node and Rust toolchains plus the Linux build deps listed under Requirements.

Run in development

# Install JS deps
npm install

# Run in dev (Vite + Tauri side-by-side)
npm run tauri dev

# Type-check frontend
npx tsc --noEmit

# Cargo check backend
cd src-tauri && cargo check

The first build downloads ~250 crates and takes 2–4 minutes; subsequent runs are seconds. The dev process opens a Tauri window and hot-reloads the React side via Vite.

Build a release

npm run tauri build

On Linux this produces a .deb, an .AppImage, and an .rpm under src-tauri/target/release/bundle/. The .deb declares nodejs as a runtime dependency (needed for the approval hook); the .AppImage is self-contained. macOS and Windows use the same command and produce .dmg / .msi bundles.

Troubleshooting

The agent never responds

Confirm the Claude Code CLI is installed and logged in: run claude in a terminal and check it answers. Agent Console can only drive a CLI that's already authenticated.

Approval modals never appear / mutations are blocked

The approval hook is a Node script — make sure Node.js ≥ 20 is on your PATH. If you installed via the .deb, the nodejs dependency should pull it in; on other distros install Node yourself.

No diff or no restore handle

The diff viewer and per-turn snapshots only work inside a git repository. Open a folder that has been git init'd (or any existing repo).

A shortcut isn't doing what I expect

Several shortcuts pass through to the shell when the terminal is focused (see the note under Terminal & sessions). Click outside the terminal to get the app-level action.

FAQ

Is this an IDE?

No. There's no code-editing UI by design — use your own editor. Agent Console is for directing and verifying an agent, not for reading and writing code by hand.

Does it send my code anywhere?

The app is local-first: no accounts, no cloud sync, no telemetry. The agent itself (Claude Code) talks to Anthropic's API exactly as the CLI does on its own — Agent Console just drives it inside your repo.

Will it touch my git branches or history?

No. Snapshots live in a private ref namespace and restore only rewrites the working tree. HEAD and your branches are never moved.

Can I let it run without approving every step?

Yes — flip the session-wide "approve all" toggle when you trust the task. The snapshot safety net still applies, so you can always rewind.

Which platforms are supported?

Linux, macOS, and Windows by design; Linux-first in practice. It's not yet packaged for Snap / Flatpak / Homebrew / the MS Store.
