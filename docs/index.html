The pitch

Most "AI coding tools" try to make agents pretend to be IDE features. Agent Console flips it: the editor is the agent, you are the lead. The center of the app is the chat, the terminal, and the git diff — not a code editor.

$_

Terminal-first

An integrated PTY shell next to the agent. Run npm test, vim, git log — anything your tools need.

▸

Agent-first

Powered by the Claude Code CLI inside your repo. Full tool access; per-tool approval on every dangerous action.

±

Diff-first

Per-turn snapshots, one-click restore, file-by-file revert. The agent edits; you verify via diff.

What's in v0.1

  • Open any local repo — folder picker, recents list
  • Integrated PTY terminal — colors, vim, htop
  • Git diff viewer — badges, revert per file, revert all
  • Chat via claude -p — stream-json bidirectional IO
  • Per-tool approval modal — Bash, Edit, Write blocked until you OK
  • Per-turn working-tree snapshot — restore with one click
  • File preview tab — diff if modified, content if clean
  • Markdown rendering — headings, lists, code blocks
  • Keyboard shortcuts — Ctrl+1/2/3, K, L, R, Esc
  • Recent projects — remembered across launches

How it works

1
Open a repo

The picker shows your recents. The app detects the project's stack (Node, Java, Rust, Python, Go…), reads the tree, and gets git status ready.

2
Direct the agent

Type a task. Behind the scenes, Agent Console builds a snapshot of the working tree and dispatches your prompt to claude -p running inside the repo.

3
Approve risky actions

When the agent wants to run Bash, Edit, or Write, a modal pops up showing the tool name and input. Approve, deny, or enable "approve all" for the session.

4
Verify via diff

Edits land in the Changes tab automatically. Click a file to see the diff. Revert per file, revert all, or restore the entire pre-turn snapshot.

5
Run anything in the terminal

Full PTY shell next to the agent — run your tests, your build, anything. Same window, same context.

Architecture

Tauri 2 binary (Rust) hosting a React frontend. Cross-process IPC for the agent via stdin/stdout. Filesystem coordination for the PreToolUse hook.

Why not VS Code?

Because VS Code is a great place to read and write code, not to direct an agent. Agent Console isn't a replacement — it's a parallel console. Keep your editor open in the next window; switch when you want to read deep. Use Agent Console for: "fix this bug", "refactor this module", "run my tests and tell me what failed", "scaffold a new endpoint and make sure it lints".
